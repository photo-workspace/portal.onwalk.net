
# 行者影像 (Onwalk) — 行走与视觉叙事的艺术

> 一个可长期生长的影像与思想档案。影像为入口，文字为结构。

[English](./README.md) | [中文版](./README_ZH.md) | [访问网站](https://www.onwalk.net)

Onwalk 是一场关于“节奏行走”与“静观实践”的长期实验。它不仅是一个摄影与写作日志的技术实现，更是一个记录城市肌理与山野幽静的影像档案，旨在探讨图像与文字如何互为骨架。

---

## 📸 内容哲学

- **行走即观看**：将行走视为一种核心的观察方式，一种缓慢的、带有身体感的视觉叙事。
- **文字骨架**：让每一段图像都有清晰的文字支撑，确保思想与画面同样具备长久的生命力。
- **极简与自主**：拒绝平台锁定。采用 Markdown 与对象存储，确保内容在数十年后依然易于迁移与归档。
- **长期主义**：关注长期的积淀而非短期的发布，让网站随时间自然生长。

## 🛠 技术架构

- **框架**: Next.js (App Router)
- **渲染**: 静态优先 (Static-first)，轻量化设计，对搜索引擎友好。
- **内容管理**: 采用去中心化思路，支持本地文件系统或 S3 兼容的对象存储。
- **资源托管**: 影像资源与应用运行环境解耦，具备高度的可移植性与 CDN 兼容性。

## 🤖 未来方向：AI 协作流

项目正尝试通过 MCP (Model Context Protocol) 协议引入 **AI 辅助工作流**。我们视 AI 为“创作伙伴”，而非作者的替代品：
- **智能整理**：自动化的元数据生成与概念化标签系统。
- **语境合成**：提供写作支持、摘要提取，并在跨度数年的档案间建立非线性联系。
- **视觉智能**：辅助发掘城市与自然景观之间隐藏的视觉模式。
